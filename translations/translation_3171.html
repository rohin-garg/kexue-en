My Thesis Writing Software Combo

By 苏剑林 | December 31, 2014

For the author, this semester's mathematical modeling class has essentially been a process of honing paper writing and Python skills. Whether it is writing papers or blog posts, I am dedicated to producing works that meet my aesthetic standards, which is why I choose $\LaTeX$ and Python. Papers produced with $\LaTeX$ are universally acknowledged to have a standard and beautiful format, and Python can indeed produce beautiful plots and concisely complete required numerical calculations. I find more and more that in mathematical modeling and writing, except for the indispensable symbolic derivation part (which can only be done with Mathematica), I can no longer do without Python.

Why demand beauty? Isn't the content enough? Indeed, content is primary, but if the presentation effect can be beautified without spending much more effort, then why not?

Using only $\LaTeX$ and Python can result in a decent paper. Of course, relying solely on these two can sometimes feel slightly insufficient. After exploring over the past month, I have chosen the following software combination as my thesis writing suite:

1. LaTeX: On Windows, I directly use CTeX as the document compilation and editing tool;

2. Python: I prefer to use version 3.4, plus a few extension modules. It can easily complete numerical calculation tasks (it "easily completes" them, rather than "completing easy" ones) and produce beautiful plots (images are exclusively in PDF vector format). Crucially, it is free and lightweight;

3. Mathematica: Symbolic computation. This goes without saying. I tried using Python's SymPy for similar work, but unfortunately, it was too slow (mainly testing symbolic integration functions);

4. Excel/Excel2LaTeX: I use Excel this way: organize data in Excel, and then use Excel2LaTeX to convert it into $\LaTeX$ code for use in $\LaTeX$. This saves a lot of work when drawing tables with large amounts of data;

5. XMind: For drawing mind maps. I have tested several mind mapping software, such as XMind, MindManager, and iMindMap, and I feel that the results produced by XMind best meet my aesthetic standards;

6. Adobe Acrobat: This is mainly used for editing PDFs and its built-in PDF printer function. Since PDFs automatically generated by software are not always good enough—for example, plots generated by Python might have excessively large white margins, or icons in maps generated by XMind might not look good or be unwanted—they can all be edited and modified with this tool.

The above is basically my software combination for paper writing. If readers have better software, feel free to suggest them~ Dedicated to creating a beautiful paper (of course, beautiful content is the most important).
